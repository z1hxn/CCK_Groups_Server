# CCK Groups – Automatic Group Assignment Logic

이 문서는 CCK 대회에서 사용되는 **자동 조편성 시스템의 동작 방식과 규칙**을 정의한다.
이 문서는 사람이 읽기 위한 설명과 동시에 **자동 구현(Codex / 개발자 구현)**을 위한 명확한 로직을 제공하는 것을 목표로 한다.

---

# 1. 역할 정의

## Player (선수 / 출전자)

- 대회에서 기록을 측정하는 참가자이다.
- 시작 시 큐브 제출대에 큐브를 제출하고 대기석에 앉는다.
- 러너의 호출을 받으면 스테이션으로 이동한다.
- 심판의 안내에 따라 기록을 측정한다.
- 기록 측정이 끝나면 **혼자 대기석으로 돌아간다.**
- 모든 기록 측정이 끝나면 경기장 밖으로 이동한다.

---

## Judge (심판)

- 스테이션을 관리하는 역할이다.
- 스테이션에 상시 착석한다.
- 러너가 데려온 선수를 1:1로 관리한다.
- 기록 측정을 감독한다.
- 기록을 입력하고 선수를 대기석으로 보낸다.

---

## Runner (러너)

- 큐브와 선수 이동을 담당한다.
- 특정 자리에 앉지 않고 계속 경기장을 돌아다닌다.

주요 작업

- 큐브 제출대의 큐브를 수집
- 큐브를 스크램블석으로 전달
- 스크램블된 큐브를 회수
- 대기석에서 선수를 호출
- 선수와 큐브를 심판 스테이션으로 전달

러너의 작업은 **정해진 순서가 아니라 상황에 따라 유동적으로 수행된다.**

---

## Scrambler (스크램블러)

- 큐브를 섞는 역할이다.
- 러너가 가져온 큐브를 스크램블한다.
- 스크램블이 완료된 큐브를 테이블에 놓으면 러너가 가져간다.

---

# 2. 대회 진행 흐름

대회는 다음 순서로 진행된다.

1. 선수가 큐브를 제출대에 제출
2. 러너가 큐브를 수집
3. 러너가 큐브를 스크램블석으로 전달
4. 스크램블러가 큐브를 섞음
5. 러너가 스크램블된 큐브를 회수
6. 러너가 큐브 ID를 보고 선수를 호출
7. 러너가 선수를 빈 스테이션으로 안내
8. 심판에게 큐브를 전달
9. 심판이 가리개를 열고 기록 측정 시작
10. 선수가 큐브를 솔빙
11. 심판이 기록을 확인하고 입력
12. 선수는 대기석으로 복귀
13. 러너가 큐브를 다시 스크램블석으로 전달

---

# 3. 조별 인원 규칙

각 **Group(조)** 에 대해 다음 인원이 설정된다.

| 역할 | 설명 |
|----|----|
| Player | 실제 기록을 측정하는 선수 |
| Judge | 스테이션 담당 심판 |
| Runner | 선수 및 큐브 이동 담당 |
| Scrambler | 큐브 스크램블 담당 |


## Player 수 결정

- 출전 인원은 이미 확정된 값이다.
- 전체 인원을 **Group 수로 나누어 최대한 균등하게 분배한다.**

예

```
전체 선수 60명
Group 4개

A = 15
B = 15
C = 15
D = 15
```


## Judge 수

- 스테이션 기반으로 결정된다.
- Player 수보다 적을 수 있다.


## Runner 수

- 보통 **2~4명**


## Scrambler 수

- 보통 **1~3명**

---

# 4. 자동 배정 로직

자동 조편성은 다음 순서로 진행된다.

```
1. Player 배정
2. Scrambler 배정
3. Runner 배정
4. Judge 배정
```

각 단계에서는 **이전에 배정된 역할과 겹치지 않도록 검사한다.**

---

# 5. 공통 제약 조건

## 1. 같은 조에서 역할 중복 금지

한 사람은 같은 Group에서

```
Player + Judge
Player + Runner
Runner + Judge
```

같이 가질 수 없다.


## 2. 조직위원 제한

조직위원(position = ADMIN)은

```
Judge
Runner
```

역할을 맡지 않는다.


## 3. Scrambler 제한

Scrambler는 **관리자가 미리 지정한 선수만 가능하다.**

---

# 6. Player 배정

1. 전체 선수 목록을 랜덤으로 섞는다.
2. Group별 Player 수에 맞게 배정한다.


## ADMIN 처리

position = ADMIN 인 선수는

**가능하면 서로 다른 Group으로 분산 배정한다.**

position 값은 다음 API로 확인한다.

```
auth.cubingclub.com/api/auth/info/{cckId}
```

---

# 7. Scrambler 배정

조건

- Scrambler 후보로 선택된 선수
- Player와 같은 Group 배정 금지

랜덤 배정한다.

---

# 8. Runner 배정

조건

- Player 아님
- Scrambler 아님
- ADMIN 아님

Group별 Runner 수에 맞게 랜덤 배정한다.

---

# 9. Judge 배정

조건

- Player 아님
- Scrambler 아님
- Runner 아님
- ADMIN 아님

Group별 Judge 수에 맞게 랜덤 배정한다.

---

# 10. 자동 조편성 실행 절차

자동 조편성 실행 시 다음 순서로 진행한다.

1. 해당 대회의 기존 조편성 데이터를 모두 삭제
2. Player 배정
3. Scrambler 배정
4. Runner 배정
5. Judge 배정
6. 결과 저장

---

# 11. 관리자 UI 흐름

자동 조편성 페이지 동작

1. 조직위원 목록 표시
2. Scrambler 후보 선수 선택
3. "자동 배정" 버튼 클릭
4. 기존 조편성 초기화 확인 모달 표시
5. 자동 배정 실행
6. 완료 메시지 표시
